## 剪切板复制失败

navigator.clipboard.writeText() 的安全上下文限制，这里我给你详细梳理一下：

1. HTTPS 与安全上下文

navigator.clipboard.writeText() 只在 安全上下文（secure context）中可用。

安全上下文包括：

HTTPS 协议下的页面 (https://)

localhost 或 127.0.0.1（开发环境特殊允许）

file:// 的一些浏览器也可能允许，但不一定可靠

简单说：在 HTTP（非 localhost）的页面上直接调用，是会失败的。

2. localhost / 127.0.0.1

浏览器会 特例允许 localhost 和 127.0.0.1 即使使用 HTTP，也算作安全上下文。

所以在开发环境中使用：

navigator.clipboard.writeText("Hello World")
  .then(() => console.log("复制成功"))
  .catch(err => console.error("复制失败", err));


是可以成功的。
